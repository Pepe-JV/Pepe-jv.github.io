Database Backup CLI Tool

Herramienta profesional de línea de comandos para copias de seguridad automáticas de múltiples sistemas de bases de datos

PHP 7.4+ CLI MySQL PostgreSQL MongoDB SQLite AWS S3 Cloud Storage

Descripción del Proyecto

Una herramienta completa de línea de comandos desarrollada en PHP para automatizar y gestionar copias de seguridad de bases de datos. Soporta múltiples sistemas de gestión de bases de datos (MySQL, PostgreSQL, MongoDB, SQLite), incluyendo características avanzadas como compresión automática, almacenamiento en la nube, notificaciones y programación automática.

Este proyecto demuestra competencias en desarrollo backend, integración de sistemas, manejo de comandos CLI, y gestión de operaciones críticas como backups y restauración de datos empresariales. Incluye arquitectura orientada a objetos con principios SOLID y manejo robusto de errores.

Características Principales

Múltiples DBMS
Soporte completo para MySQL, PostgreSQL, MongoDB y SQLite con validación automática de conexiones antes de cada operación
Tipos de Backup
Backups completos, incrementales y diferenciales con compresión gzip automática para optimizar espacio
Almacenamiento Cloud
Integración directa con AWS S3, Google Cloud Storage y Azure Blob Storage para backup remoto seguro
Logging Completo
Sistema de logs detallado con niveles, salida colorizada en terminal y registro de todas las métricas
Notificaciones
Notificaciones automáticas vía Slack o Email al completar operaciones, incluyendo estadísticas detalladas
Restauración Rápida
Restauración completa con descompresión automática y validación de integridad de archivos

Stack Tecnológico

PHP 7.4+ MySQL Client (mysqldump) PostgreSQL Client (pg_dump) MongoDB Tools (mongodump) AWS CLI Google Cloud SDK Azure CLI gzip Compression JSON Config Shell Commands

Ejemplos de Uso

Crear Backup Completo

bash 
php backup.php backup --config=config.json --type=full --cloud --notify

Probar Conexión a Base de Datos

bash 
php backup.php test-connection --config=config.json

Restaurar Base de Datos

bash 
php backup.php restore --config=config.json --file=backups/midb_full_2024-12-24.sql.gz

Listar Backups Disponibles

bash 
php backup.php list-backups --config=config.json

Configuración

El sistema utiliza un archivo JSON para toda la configuración, facilitando la gestión de múltiples entornos:

json 
{
  "database": {
    "type": "mysql",
    "host": "localhost",
    "port": 3306,
    "username": "root",
    "password": "password",
    "database": "mi_base_de_datos"
  },
  "backup": {
    "directory": "backups",
    "compress": true,
    "type": "full"
  },
  "cloud_storage": {
    "enabled": true,
    "provider": "s3",
    "bucket": "mi-bucket-backups",
    "region": "us-east-1"
  },
  "notifications": {
    "enabled": true,
    "type": "slack",
    "webhook_url": "https://hooks.slack.com/services/..."
  }
}

Casos de Uso

  • Backups Automáticos Programados: Configurar con cron para backups nocturnos automáticos de producción
  • Pre-Deployment Safety: Crear snapshot antes de actualizar aplicaciones en producción para rollback rápido
  • Disaster Recovery: Restauración rápida y confiable en caso de fallos o corrupción de datos
  • Migración de Servidores: Facilitar el traspaso de bases de datos entre entornos (dev, staging, prod)
  • Auditoría y Compliance: Mantener histórico de backups con logging detallado para cumplir regulaciones
  • Desarrollo y Testing: Crear copias de bases de datos de producción para entornos de desarrollo seguros

Seguridad y Buenas Prácticas

  • Validación de Credenciales: Verificación antes de operaciones críticas
  • Manejo Robusto de Errores: Logs detallados y mensajes informativos
  • Compresión Automática: Optimización de espacio de almacenamiento
  • Variables de Entorno: Soporte para credenciales sensibles
  • Encriptación en Tránsito: Seguridad al subir a cloud storage
  • Permisos de Archivos: Sistema configurable con chmod/chown
  • Auditoría Completa: Logging de todas las operaciones

Arquitectura del Sistema

El proyecto está organizado en clases especializadas siguiendo principios SOLID para mantener código limpio y mantenible:

  • Logger: Sistema centralizado de logging con niveles (INFO, WARNING, ERROR, SUCCESS) y salida colorizada
  • DatabaseConnector: Abstracción de conexiones para múltiples DBMS con testing integrado
  • BackupManager: Orquestación de operaciones de backup y restauración con validación de integridad
  • CloudStorage: Integración con proveedores de almacenamiento cloud usando SDKs oficiales
  • Notifier: Sistema de notificaciones multi-canal (Slack, Email) con mensajes formateados

Aprendizajes Clave del Proyecto

  • Integración con servicios cloud (AWS, GCP, Azure)
  • Gestión de errores y logging profesional

  • Automatización de tareas críticas de infraestructura
  • Manejo seguro de credenciales y datos sensibles
  • Testing de conexiones y validación pre-operación
